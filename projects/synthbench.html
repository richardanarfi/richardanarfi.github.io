SynthEval

Hand-authoring evaluation tasks for tool-using agents is expensive and slow. SynthEval generates synthetic tasks at scale, validates them with a multi-judge panel, and produces ground truth that's statistically indistinguishable from human-authored data.

Motivation

The Problem

Human authoring doesn't scale

Writing a single high-quality evaluation task (with a query, expected tool calls, gold answer, difficulty label, and rubric) takes 15-30 minutes. A benchmark suite of 200 tasks across multiple categories requires weeks of expert time. Every time the tool set changes, the benchmark needs new tasks.

Naive generation produces degenerate tasks

Simply prompting an LLM to "generate evaluation tasks" yields tasks that are too easy, too similar to each other, or not representative of real usage. Without validation, synthetic benchmarks give false confidence because models ace them but fail on real-world queries.

Human Seed Tasks

The pipeline starts with 15-20 hand-authored seed tasks spanning difficulty levels (easy, medium, hard) and tool categories (retrieval, calculation, multi-step, error recovery). Each seed is a complete evaluation record, not just a query, but the full expected trace.

JSON 
{
  "seed_id": "seed_003",
  "category": "retrieval",
  "difficulty": "medium",
  "query": "Find the current CEO of Anthropic and their educational background",
  "expected_tools": [
    {"tool": "web_search", "params": {"query": "Anthropic CEO"}}
  ],
  "gold_answer": "Dario Amodei, PhD in Computational Neuroscience from Princeton",
  "reasoning_trace": "Single retrieval task requiring entity extraction from search results",
  "domain_tags": ["company_info", "person_lookup"],
  "distractor_tools": ["database_query", "calculator"]
}

The distractor_tools field lists tools that are plausible but wrong for this task. During generation, the pipeline creates variants that test whether the agent resists these distractors.

Three Expansion Strategies

Each seed task is expanded using three complementary strategies. The generator uses few-shot prompting with the seed as an example, plus 2-3 other seeds for diversity.

  • Paraphrase: Same task, different wording. "Find the CEO of Anthropic" becomes "Who leads Anthropic as chief executive?" Tests whether the agent is robust to surface variation.
  • Analogize: Same structure, different domain. A CEO lookup becomes a CTO lookup for a different company. Tests whether the agent generalizes the pattern.
  • Compose: Combine two seed tasks into a multi-step task. A retrieval seed + a calculation seed becomes "Find Anthropic's founding year and calculate how many years ago that was."

Quality Filter

Generated tasks pass through three filters before acceptance:

  1. Deduplication: Embedding cosine similarity > 0.92 with any existing task → reject
  2. Self-consistency: Run the task through a model twice. If answers diverge, the task is ambiguous → reject
  3. Difficulty calibration: If a "hard" task is solved correctly by all 3 test models, downgrade to "medium"
Python 
def quality_filter(task, existing_tasks, embedder, models):
    """Three-stage quality filter for generated tasks."""
    # 1. Deduplication via embedding similarity
    task_embedding = embedder.encode(task.query)
    for existing in existing_tasks:
        sim = cosine_similarity(task_embedding, existing.embedding)
        if sim > 0.92:
            return Reject("too_similar", similarity=sim)

    # 2. Self-consistency check
    answers = [run_task(model, task) for model in models[:2]]
    if not semantically_equivalent(answers[0], answers[1]):
        return Reject("ambiguous", answers=answers)

    # 3. Difficulty calibration
    pass_count = sum(1 for m in models if run_task(m, task).correct)
    if task.difficulty == "hard" and pass_count == len(models):
        task.difficulty = "medium"

    return Accept(task)

Multi-Judge Ground Truth Validation

Every generated task is validated by a 3-judge panel. Each judge independently evaluates whether the expected tool calls and gold answer are correct for the query. Using different models as judges provides reasoning diversity, and agreement across models is a strong signal of correctness.

Python 
async def validate_ground_truth(task, judges):
    """Three-judge validation with debate resolution."""
    # Run all judges in parallel
    verdicts = await asyncio.gather(*[
        judge.evaluate(task) for judge in judges
    ])
    # judges = [claude-opus, claude-sonnet, claude-haiku]

    relevant_indices = [v.relevant_record_indices for v in verdicts]

    # Check agreement per record
    for record_idx in all_record_indices(task):
        votes = [record_idx in ri for ri in relevant_indices]
        if all(votes):       # Unanimous relevant → include
            final.include(record_idx)
        elif not any(votes): # Unanimous not relevant → exclude
            final.exclude(record_idx)
        else:                # Disagreement → debate
            resolution = await debate_arbiter.resolve(
                task, record_idx, verdicts
            )
            final.apply(resolution)

    return final

Sentinel Validation

After ground truth is finalized, a sentinel check verifies that every task's expected records are present in the ground truth. If a seed task's expected tool calls don't appear as relevant in the final ground truth, something went wrong in generation or validation. All sentinel scores must be 1.0 — any failure triggers re-evaluation.

Automated Rubric Generation

Each task gets an auto-generated rubric with 3-5 evaluation criteria. Rubrics support alternate-valid tool call sequences, since there may be multiple correct ways to solve a task, and the rubric should accept all of them.

Synthetic vs human task difficulty ranking

Synthetic tasks match human discriminative power

The synthetic benchmark separated strong and weak models with the same ranking as the human-authored benchmark (Spearman ρ = 0.94). The key was difficulty calibration; without it, synthetic tasks clustered at "easy" and lost discriminative power.

Composition creates the hardest tasks

Tasks generated via the "compose" strategy (combining two seeds) had the highest failure rate across all models (62% vs 23% for paraphrase and 31% for analogize). Composition is where synthetic generation adds the most value because these tasks are hard to write by hand.

Insights

Lessons Learned

Without seeds, LLMs generate tasks that are superficially diverse but structurally identical: different topics but the same retrieval pattern, the same difficulty, and the same expected tool. The seed-based approach forces structural diversity by design: each seed defines a different task skeleton.

The first pass of generation produced 78% "easy" tasks and only 8% "hard" tasks. Models aced the benchmark and it provided no signal. After adding the calibration step (if all models pass a "hard" task, downgrade it), the distribution shifted to 35/42/23 easy/medium/hard. This matched the human-authored distribution and restored discriminative power.

Claude Haiku was consistently more lenient, marking 12% more records as relevant than Claude Opus. Using a single judge would have produced systematically inflated ground truth. The three-judge panel with debate resolution produced ground truth that matched human annotations at 96% agreement, versus 89% for single-judge validation.
