Tool call evaluation framework

ToolEval

A framework for evaluating how LLM agents select, parameterize, and compose tool calls. Measures tool selection accuracy, parameter correctness, regression when new tools are added, and interaction effects between tools using Shapley-based metrics.

  • Three-step pipeline: baseline, perturbation, forward check
  • Metrics: tool regression score, dominance index, routing entropy
  • LLM-as-judge panel for synthesis quality and tool bleed

Synthetic evaluation data pipeline

SynthEval

A pipeline for generating synthetic evaluation tasks for tool-using agents. Expands hand-authored seed tasks via paraphrase, analogy, and composition strategies, then validates with a multi-judge panel for ground truth quality.

  • Three expansion strategies with embedding-based dedup
  • Three-judge validation with debate resolution
  • Sentinel validation for 100% ground truth recall

Prompt engineering & evaluation

PromptLab

A framework for systematic prompt iteration: regression testing against golden sets, structural decomposition of failures, LLM-as-judge triage, and automated ground truth evolution that never loses valid alternatives.

  • Regression runner with structural diff decomposition
  • Three-judge panel (opus/sonnet/haiku) with debate arbiter
  • Automated ground truth updates with full audit trail

Reinforcement learning for LLMs

GRPO Arithmetic

An exploration of Group Relative Policy Optimization for teaching arithmetic to a small language model. Investigates reward design, generalization gaps, and how multi-signal reward shaping prevents output collapse.

  • GRPO training with composite reward functions
  • Reward shaping to prevent reward hacking
  • In-domain vs out-of-domain generalization analysis

About

About

Applied scientist building evaluation-first LLM systems, from tool-calling agents to RL-trained models.

Background

I'm an Applied Scientist working on machine learning-driven agent systems at scale. My day-to-day involves building evaluation pipelines, designing metrics for agent reliability, and turning messy real-world signals into structured, measurable behaviors. I hold a Ph.D. in Computer Science and have experience spanning deep learning, reinforcement learning, and production LLM systems.

What I care about

I believe the hardest problem in LLM engineering isn't making models more capable - it's knowing whether they actually work. I focus on evaluation frameworks that go beyond pass/fail, reward design that resists hacking, and prompt engineering workflows with the same rigor as software testing. If you can't measure it, you can't ship it.
